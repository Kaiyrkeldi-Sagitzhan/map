## Общая архитектура системы

Проект реализован на основе **микросервисной архитектуры**, что обеспечивает:
- Независимое масштабирование компонентов
- Изоляцию сбоев
- Гибкость в выборе технологий для каждого сервиса
- Удобство разработки и тестирования

### Структура микросервисов:

```
┌─────────────────────────────────────────────────────────────────┐
│                        Docker Network                            │
│                    (kzmap-network)                               │
│                                                                 │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐         │
│  │  PostgreSQL │    │    Redis    │    │  Nginx GW   │         │
│  │  + PostGIS  │◄───┤   (Cache)   │◄───┤ (API Gateway)│         │
│  └─────┬───────┘    └─────┬───────┘    └─────┬───────┘         │
│        │                  │                  │                 │
│        │                  │                  │                 │
│  ┌─────▼───────┐    ┌─────▼───────┐    ┌─────▼───────┐         │
│  │ Auth Service│    │ Map Service │    │ Webhook Svc │         │
│  │   (Go)      │    │    (Go)     │    │    (Go)     │         │
│  └─────────────┘    └─────────────┘    └─────────────┘         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Технологический стек backend:
- **Язык программирования:** Go 1.21
- **Веб-фреймворк:** Gin (Gin-gonic) v1.9.1
- **База данных:** PostgreSQL 15 + PostGIS 3.3
- **Кэширование:** Redis 7-alpine
- **Оркестрация:** Docker Compose
- **Протокол:** HTTP/REST API с JWT-аутентификацией

## Backend-сервисы

### Auth Service

**Назначение:** Обработка аутентификации, авторизации и управления пользователями.

**Основные компоненты:**

#### 1. Конфигурация (`internal/config/config.go`)
- Загрузка переменных окружения
- Настройки подключения к БД
- Параметры JWT и OAuth
- Настройки SMTP для email-уведомлений

#### 2. Модели (`internal/model/user.go`)
```go
type User struct {
    ID        string    `json:"id" db:"id"`
    Email     string    `json:"email" db:"email"`
    Password  string    `json:"-" db:"password_hash"`
    Role      string    `json:"role" db:"role"`
    FirstName string    `json:"firstName" db:"first_name"`
    LastName  string    `json:"lastName" db:"last_name"`
    Nickname  string    `json:"nickname" db:"nickname"`
    CreatedAt time.Time `json:"createdAt" db:"created_at"`
    UpdatedAt time.Time `json:"updatedAt" db:"updated_at"`
}
```

#### 3. Репозиторий (`internal/repository/user_repository.go`)
- CRUD-операции с пользователями
- Поиск по email
- Управление ролями

#### 4. Сервисы:
- **AuthService:** Основная логика аутентификации
- **EmailService:** Отправка email-уведомлений через SMTP
- **OAuthService:** Интеграция с Google OAuth 2.0
- **VerificationService:** Подтверждение email и верификация

#### 5. Хэндлеры (`internal/handler/auth_handler.go`)
Эндпоинты:
- `POST /auth/register` - Регистрация пользователя
- `POST /auth/login` - Аутентификация (JWT токен)
- `POST /auth/logout` - Выход
- `POST /auth/refresh` - Обновление токена
- `POST /auth/verify-email` - Подтверждение email
- `POST /auth/forgot-password` - Восстановление пароля
- `POST /auth/google` - Аутентификация через Google OAuth
- `GET /auth/me` - Получение профиля текущего пользователя

**Особенности реализации:**
- Использование bcrypt для хеширования паролей
- JWT-токены с валидацией и автоматическим обновлением
- Rate limiting для защиты от брутфорса
- Валидация входных данных через go-playground/validator

### Map Service

**Назначение:** Управление географическими объектами, версионирование, история изменений, работа с жалобами.

**Основные компоненты:**

#### 1. Модели данных:

**GeoObject (`internal/model/geo_object.go`):**
```go
type GeoObject struct {
    ID          string         `json:"id" db:"id"`
    BaseID      string         `json:"baseId" db:"base_id"`
    Version     int            `json:"version" db:"version"`
    OwnerID     string         `json:"ownerId" db:"owner_id"`
    Scope       string         `json:"scope" db:"scope"` // global, private
    Type        string         `json:"type" db:"type"`   // river, lake, mountain, city, road, boundary, forest, building
    Name        string         `json:"name" db:"name"`
    Description string         `json:"description" db:"description"`
    Metadata    postgres.Jsonb `json:"metadata" db:"metadata"`
    Geometry    postgres.Jsonb `json:"geometry" db:"geometry"`
    CreatedAt   time.Time      `json:"createdAt" db:"created_at"`
    UpdatedAt   time.Time      `json:"updatedAt" db:"updated_at"`
}
```

**Важность PostGIS:**
- Хранение геометрии в формате WKB/WKT
- Поддержка пространственных индексов GIST
- Выполнение пространственных запросов (ST_Within, ST_Intersects и др.)
- Экспорт в GeoJSON через ST_AsGeoJSON

#### 2. Система версионирования (`geo_object_version_service.go`)
- Каждое изменение создает новую версию объекта
- Сохранение истории всех изменений
- Возможность отката к предыдущим версиям
- База_id для группировки всех версий одного объекта

#### 3. История изменений (`geo_object_history_service.go`)
- Фиксация всех действий (create, update, delete)
- Сохранение снапшотов до и после изменений
- Привязка изменений к пользователям
- Хранение в формате JSONB для гибкости

#### 4. Репозитории:
- `GeoObjectRepository` - основные операции с объектами
- `GeoObjectHistoryRepository` - работа с историей
- `GeoObjectVersionRepository` - управление версиями
- `ComplaintRepository` - система жалоб
- `RedisCache` - кэширование запросов

#### 5. Сервисы:
- **GeoObjectService:** Бизнес-логика работы с объектами
- **GeoObjectHistoryService:** Управление историей изменений
- **GeoObjectVersionService:** Логика версионирования
- **ComplaintService:** Обработка жалоб пользователей

#### 6. Хэндлеры (`internal/handler/*.go`)

**Основные эндпоинты Map Service:**

```
GET    /health                          - Проверка здоровья сервиса

# Географические объекты
GET    /api/geo-objects                 - Получение списка (с фильтрацией)

# Версии
GET    /api/geo-objects/:id/versions    - Список версий объекта

# История
GET    /api/geo-objects/:id/history     - История изменений объекта

# Жалобы
POST   /api/complaints                  - Создание жалобы
GET    /api/complaints                  - Список жалоб (с фильтрацией)
GET    /api/complaints/:id              - Детали жалобы
PUT    /api/complaints/:id/status       - Обновление статуса жалобы

# Кэширование
POST   /api/cache/invalidate             - Инвалидация кэша
```

**Особенности реализации:**
- Пагинация для больших списков объектов
- Фильтрация по scope, type, owner_id
- Пространственные фильтры (в радиусе, в границах)
- Оптимизированные SQL-запросы с JOIN
- Автоматическое управление транзакциями

## Базы данных и хранение данных

### PostgreSQL + PostGIS

**Схема базы данных включает следующие таблицы:**

#### 1. `users` - Пользователи
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL DEFAULT 'user',
    first_name VARCHAR(100) DEFAULT '',
    last_name VARCHAR(100) DEFAULT '',
    nickname VARCHAR(100) DEFAULT '',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**Индексы:**
- Индекс на email (UNIQUE)
- Композитный индекс для поиска по имени

#### 2. `geo_objects` - Географические объекты
```sql
CREATE TABLE geo_objects (
    id VARCHAR(50) PRIMARY KEY,
    base_id UUID NOT NULL,
    version INTEGER NOT NULL DEFAULT 0,
    owner_id UUID REFERENCES users(id) ON DELETE SET NULL,
    scope VARCHAR(20) NOT NULL CHECK (scope IN ('global', 'private')),
    type VARCHAR(50) NOT NULL,
    name VARCHAR(255) NOT NULL,
    description TEXT,
    metadata JSONB DEFAULT '{}',
    geometry GEOMETRY(GEOMETRY, 4326) NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**Пространственные индексы:**
- GIST индекс на колонку geometry для пространственных запросов

**Обычные индексы:**
- scope, type (для фильтрации слоев)
- owner_id (для фильтрации по владельцу)
- Композитный индекс (scope, owner_id, type)
- metadata (GIN индекс для JSONB запросов)

**Триггеры:**
- Автообновление updated_at при изменении записи

#### 3. `geo_object_history` - История изменений
```sql
CREATE TABLE geo_object_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    object_id UUID NOT NULL,
    user_id UUID NOT NULL,
    action VARCHAR(20) NOT NULL CHECK (action IN ('create', 'update', 'delete')),
    description TEXT NOT NULL,
    before_snapshot JSONB,
    after_snapshot JSONB,
    created_at TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);
```

**Индексы:**
- object_id (для поиска истории конкретного объекта)
- user_id (для поиска действий пользователя)
- created_at DESC (для хронологического отображения)

#### 4. `geo_object_versions` - Система версионирования
```sql
CREATE TABLE geo_object_versions (
    id VARCHAR(50) PRIMARY KEY,
    base_id UUID NOT NULL,
    version INTEGER NOT NULL,
    object_id VARCHAR(50) NOT NULL,
    -- остальные поля geo_objects
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**Ограничения:**
- UNIQUE (base_id, version) - гарантия уникальности версий

#### 5. `complaints` - Жалобы пользователей
```sql
CREATE TABLE complaints (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    object_id UUID REFERENCES geo_objects(id) ON DELETE SET NULL,
    object_type VARCHAR(50) NOT NULL,
    description TEXT NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'pending' 
        CHECK (status IN ('pending', 'in_review', 'resolved', 'dismissed')),
    admin_notes TEXT DEFAULT '',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**Индексы:**
- user_id, object_id, status

#### 6. Представления (Views)

**`accessible_geo_objects`:**
```sql
CREATE OR REPLACE VIEW accessible_geo_objects AS
SELECT 
    id, owner_id, scope, type, name, description, metadata, 
    ST_AsGeoJSON(geometry) as geometry_json, 
    created_at, updated_at
FROM geo_objects
WHERE scope = 'global' 
   OR owner_id = current_setting('app.current_user_id', true)::uuid
   OR current_setting('app.current_user_role', true) = 'admin';
```

Позволяет эффективно фильтровать доступные пользователю объекты на уровне БД.

### Redis

**Назначение:**
- Кэширование результатов пространственных запросов
- Кэширование списка объектов по типам и областям
- Хранение сессий и временных данных
- Очереди задач (при необходимости)

**Конфигурация:**
- Maxmemory: 512MB
- Политика вытеснения: allkeys-lru
- Сетевые настройки: разрешен доступ из Docker сети

**Ключи кэширования:**
- `geo_objects:{type}:{bounds}` - объекты по типу и границам
- `geo_object:{id}` - конкретный объект
- `user:{id}:objects` - объекты пользователя
- `stats:{type}` - статистика по типам объектов

**Стратегия инвалидации:**
- При создании/обновлении/удалении объекта
- При изменении прав доступа
- Периодическая (TTL 1 час для часто запрашиваемых данных)

### Безопасность сети

**Изоляция:**
- Внутренняя сеть недоступна извне

- Сервисы общаются через внутренние DNS-имена

**Firewall правила:**
- Входящие: только 80, 443 (HTTP/HTTPS)
- Межсервисный: через Docker сеть
- Исходящие: ограничено (SMTP, OAuth)

**Межсервисная аутентификация:**
- JWT-токены для API запросов
- Проверка подписи и валидации токенов
- Ролевая модель (admin, expert, user)

## Безопасность и аутентификация

### JWT Аутентификация

**Реализация в Go:**
```go
type TokenManager struct {
    secretKey   []byte
    expiryHours int
}

func (tm *TokenManager) GenerateToken(userID string, role string) (string, error) {
    claims := jwt.MapClaims{
        "user_id": userID,
        "role":    role,
        "exp":     time.Now().Add(time.Duration(tm.expiryHours) * time.Hour).Unix(),
        "iat":     time.Now().Unix(),
    }
    
    token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)
    return token.SignedString(tm.secretKey)
}
```

**Валидация middleware:**
```go
func AuthMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        tokenString := extractToken(c)
        token, err := jwt.Parse(tokenString, func(token *jwt.Token) (interface{}, error) {
            return []byte(os.Getenv("JWT_SECRET")), nil
        })
        
        if err != nil || !token.Valid {
            c.AbortWithStatusJSON(401, gin.H{"error": "unauthorized"})
            return
        }
        
        // Сохранение claims в context
        c.Set("user_id", claims["user_id"])
        c.Set("role", claims["role"])
        c.Next()
    }
}
```

```go
func (s *GeoObjectVersionService) CreateVersion(ctx context.Context, objectID string, changes UpdateObjectInput) (*GeoObject, error) {
    // Получение текущей версии
    current, err := s.repo.GetByID(ctx, objectID)
    if err != nil {
        return nil, err
    }
    
    // Создание новой версии
    newVersion := &GeoObject{
        ID:        uuid.New().String(),
        BaseID:    current.BaseID,
        Version:   current.Version + 1,
        OwnerID:   current.OwnerID,
        Scope:     current.Scope,
        Type:      changes.Type,
        Name:      changes.Name,
        Geometry:  changes.Geometry,
        Metadata:  changes.Metadata,
    }
    
    // Сохранение новой версии
    if err := s.repo.Create(ctx, newVersion); err != nil {
        return nil, err
    }
    
    // Архивация старой версии в таблицу версий
    s.archiveVersion(ctx, current)
    
    return newVersion, nil
}
```

## Оптимизация и кэширование

### Стратегия кэширования

**Что кэшируется:**
1. Списки объектов по типам и географическим границам
2. Детальная информация об объектах
3. Статистика (количество объектов по типам)
4. Результаты пространственных запросов

**TTL (Time To Live):**
- Статические данные (типы объектов): 24 часа
- Динамические данные (списки объектов): 1 час
- Детальная информация: 30 минут
- Пространственные запросы: 15 минут

**Кэширование пространственных запросов:**
```go
func (s *GeoObjectService) GetObjectsInBounds(ctx context.Context, bounds Bounds) ([]*GeoObject, error) {
    // Генерация ключа кэша
    cacheKey := fmt.Sprintf("geo:bbox:%f:%f:%f:%f", 
        bounds.MinLng, bounds.MinLat, bounds.MaxLng, bounds.MaxLat)
    
    // Попытка получить из кэша
    if cached, err := s.cache.Get(ctx, cacheKey); err == nil {
        var objects []*GeoObject
        json.Unmarshal(cached, &objects)
        return objects, nil
    }
    
    // Если нет в кэше - запрос к БД
    objects, err := s.repo.GetByBounds(ctx, bounds)
    if err != nil {
        return nil, err
    }
    
    // Сохранение в кэш
    data, _ := json.Marshal(objects)
    s.cache.Set(ctx, cacheKey, data, time.Hour)
    
    return objects, nil
}
```
