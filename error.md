# Ошибки сборки проекта (ветка ask)

Ниже описаны критические ошибки, блокирующие сборку Docker-контейнеров, и причины их возникновения.

## 1. Бэкенд (map-service)
**Файл:** `map-service/internal/repository/geo_object_repository.go`
**Ошибки:** 
- Строка 544, 595 и др.: `syntax error: unexpected comma, expected name`
- Строка 573, 637 и др.: `syntax error: unexpected type, expected expression`

### Причина:
В коде методов `UpdateFromSnapshot` и `RestoreFromSnapshot` используется переменная с именем `type`. В языке Go `type` является зарезервированным ключевым словом и не может использоваться в качестве имени переменной.

### Рекомендация по фиксу:
Переименовать переменную `type` во всех методах (например, в `objType`). Также следует исправить извлечение данных из `map[string]interface{}`, так как поля `geometry` и `metadata` требуют корректного приведения типов или повторной маршализации в JSON.

## 2. Фронтенд (frontend)
**Файл:** `frontend/src/components/Editor/LayersPanel.tsx`
**Ошибка:** `error TS6133: 'apiService' is declared but its value is never read.`

### Причина:
В файле импортирован `apiService`, но он нигде не используется в коде компонента. В проекте настроена строгая проверка TypeScript (no-unused-vars), которая блокирует сборку при наличии неиспользуемых импортов.

### Рекомендация по фиксу:
Удалить строку импорта `apiService`.

---

## 3. Проблема с историей (Логика)
**Статус:** История не сохраняется для импортированных объектов.

### Причина:
Текущая система истории реализована через триггеры на таблице `geo_objects`. Однако многие объекты на карте подгружаются через векторные тайлы и физически отсутствуют в таблице `geo_objects` до момента их первого сохранения через редактор. При попытке обновления (`Update`) таких объектов бэкенд возвращает `ErrObjectNotFound`, и триггер `UPDATE` не срабатывает.

### Попытка решения (отменена пользователем):
Была предложена Upsert-логика в `GeoObjectService.Update`: если объект не найден по ID, выполнять `Create`, чтобы объект появился в базе и для него началась история (триггер `INSERT`). Сеньор-программисту нужно реализовать это или аналогичный механизм синхронизации.
