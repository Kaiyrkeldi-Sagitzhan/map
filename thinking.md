## ЧАСТЬ 2: КОНЦЕПЦИЯ ИНТЕРФЕЙСА — "Умная Панель Свойств"

Мы реализуем **динамическую адаптивную форму**, которая меняет свой состав полей в зависимости от категории (`fclass`) объекта.

### 2.1 Динамические формы по категориям:

| Категория | Визуальный стиль | Уникальные поля (Metadata JSONB) |
| :--- | :--- | :--- |
| **⛰️ Горы (Peak)** | Высотная форма | Высота (м), Сложность, Тип вершины, Наличие ледника. |
| **🏢 Здания (Building)** | Архитектурная форма | Этажность, Материал, Год постройки, Износ (%). |
| **🌊 Вода (Water)** | Гидрологическая форма | Глубина (м), Прозрачность, Скорость течения, Тип дна. |
| **🏥 Соц. объекты** | Реестровая форма | Вместимость, Время работы, Контакты, Ответственное лицо. |
| **🛣️ Дороги (Roads)** | Транспортная форма | Тип покрытия, Ограничение скорости, Состояние полотна. |

### 2.2 Типы элементов управления в формах:
*   **Слайдеры:** Для быстрой установки этажности или глубины.
*   **Выпадающие списки:** Для выбора материала (Кирпич, Панель) или сложности гор.
*   **Индикаторы:** Визуальные иконки (`lucide-react`) для каждого параметра.

## ЧАСТЬ 3: ГЛОБАЛЬНЫЙ ПОИСК И АНАЛИЗ (Search Area)

Мы внедряем функционал **"Поиск в области"**, позволяющий массово находить и анализировать объекты на карте.

### 3.1 Функции поиска:
*   **Выделение области:** Пользователь рисует прямоугольник на карте.
*   **Фильтрация по атрибутам:**
    *   "Найти все горы выше 3000м в этом районе".
    *   "Показать все школы с вместимостью более 500 мест в радиусе 10км".
    *   "Выделить все мосты и тоннели на выбранной трассе".
*   **Массовый экспорт:** Возможность выгрузить список найденных объектов в CSV/Excel.

## ЧАСТЬ 4: ТЕХНИЧЕСКИЙ ПЛАН РЕАЛИЗАЦИИ

1.  **Этап "Схемы" (`schema.ts`):** Описание 100+ маппингов `fclass -> fields[]`. Каждому типу объекта — своя уникальная форма.
2.  **Этап "Поиск" (Backend/PostGIS):** Реализация пространственных запросов (`ST_Contains`) для поиска объектов внутри выделенной области.
3.  **Этап "Интерфейс" (`PropertiesPanel.tsx`):** Переработка панели свойств в динамический конструктор.
4.  **Этап "Интеграция":** Подгрузка существующих данных из `kazakhstan.gpkg` в новые формы (население, скорость и т.д.).
